# 面試展示講稿

## 一句話介紹

這是一個遊戲商城 demo，使用 Next.js 做前台互動，Express 提供會員、購物車、訂單與付款 API，並用 SQLite 保存本機展示資料。

## 展示流程

1. 從首頁瀏覽遊戲，選商品加入購物車。
2. 進入購物車填寫收件資料並送出結帳。
3. 訂單中心會出現待付款訂單。
4. 使用 Stripe 測試卡或 Demo 快速付款完成付款。
5. 訂單狀態更新為付款完成。
6. 管理員登入後可在後台查看訂單與商品管理。

## 前後端分工

前端負責商品列表、購物車、訂單中心、付款畫面、願望清單與管理後台。後端負責登入驗證、訂單狀態、付款資料建立、SQLite persistence 與檔案上傳。

## 付款流程

正式信用卡流程是前端載入 Stripe 的測試付款元件，後端用 `STRIPE_SECRET_KEY` 建立付款資料。付款成功後，前端通知後端確認付款結果，訂單狀態更新為 `paid`，畫面顯示「付款完成」。

如果 Stripe key 沒有設定，畫面會提供 Demo 快速付款，讓面試展示仍可完整跑完購物流程。

## AI / RAG

專案有 AI/chat 入口，可用來展示客服或商品問答方向。正式化時可以把商品、訂單 FAQ、退換貨規則整理成可檢索資料，讓 AI 回答更貼近商城情境。

## Demo persistence

目前使用 SQLite 保存資料，適合本機展示與面試講解。正式部署會改成雲端資料庫，並加入 migration、備份、監控與權限管理。

## 正式化改進方向

- SQLite 改成 PostgreSQL 或 MySQL。
- local upload 改成 Vercel Blob、S3 或 Cloudinary。
- 管理員密碼改成正式環境變數與更完整的角色權限。
- Stripe 加上 webhook 驗證，避免只依賴前端回報付款結果。
- 補完整 e2e 測試，涵蓋登入、結帳、付款、訂單狀態與管理後台。
- AI/chat 加上可觀測性、資料更新流程與安全限制。
